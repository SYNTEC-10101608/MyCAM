## 1.1 軟件概述

**MyCAM** 是一款專門用於五軸激光切割加工設計的 CAD/CAM 集成軟件系統。該軟件結合了計算機輔助設計（CAD）和計算機輔助制造（CAM）功能，提供從 3D 模型導入、路徑規劃、姿態調整到 NC 程序輸出的完整解決方案。

### 核心技術基礎

**MyCAM** 基於以下核心技術構建：

| 技術組件 | 版本/說明 | 用途 |
|---------|----------|------|
| **.NET Framework** | 4.8 | 軟件運行平台 |
| **C# 語言** | 7.3 | 主要開發語言 |
| **OpenCascade** | 3D 幾何內核 | 3D 建模與幾何處理 |
| **VNC 協議** | TightVNC 兼容 | 遠程連接與監控 |

軟件采用 OpenCascade 幾何內核，提供強大的 3D 幾何處理能力，使用戶能夠精確地進行加工路徑設計和姿態調整。

## 1.2 軟件用途與應用場景

### 主要功能模塊

**MyCAM** 軟件主要提供以下核心功能：

**1. CAD 功能**
- 3D 模型導入與處理（支持 STEP、IGES、STL 格式）
- 模型縫補與檢查
- 坐標系轉換與調整
- 特徵點與特徵線創建

**2. CAM 功能**
- 加工路徑生成（支持線段、圓弧等多種幾何類型）
- 姿態調整與優化
- 路徑排序與管理
- 引線、過切、微連設置
- NC 程序輸出

**3. 仿真功能**
- 實時加工仿真
- 碰撞檢測
- 加工過程可視化

**4. 遠程監控**
- VNC 遠程連接
- 實時數據傳輸

### 應用場景

**MyCAM** 適用於以下加工領域：

| 應用類型 | 說明 | 典型工件 |
|---------|------|---------|
| **異形蓋件** | 五軸聯動激光切割 | 汽車覆蓋件、飛機蒙皮 |
| **管拉件** | 管材自動化路徑生成 | 排氣管、結構管材 |
| **復雜曲面** | 精密空間曲面加工 | 模具、雕刻件 |
| **批量生產** | 多工件自動化排產 | 標準件批量加工 |

## 1.3 系統需求

### 1.3.1 硬件需求

| 硬件 | 最低需求 | 推薦配置 |
|------|----------|----------|
| **處理器** | Intel Core i5 或同級別 | Intel Core i7 或更高 |
| **內存** | 8 GB RAM | 16 GB RAM 或更高 |
| **顯卡** | 支持 OpenGL 3.3 以上 | 專業顯卡，支持 OpenGL 4.5 |
| **硬盤空間** | 5 GB 可用空間 | 10 GB 以上（含項目文件空間）|
| **屏幕分辨率** | 1920 × 1080 | 2560 × 1440 或更高 |
| **網絡** | 可選（用於 VNC 連接）| 100 Mbps 以上有線網絡 |

### 1.3.2 軟件需求

| 項目 | 需求 |
|------|------|
| **操作系統** | Windows 11（64 位版）|
| **.NET Framework** | .NET Framework 4.8 或更高版本 |
| **OpenCascade** | 已集成到軟件（軟件內置）|
| **VNC Viewer** | TightVNC 或兼容軟件（用於遠程連接功能）|

### 1.3.3 其他需求

**授權認證：**
- 軟件啟動時需通過授權驗證
- 如出現授權錯誤，請聯系系統管理員

**VNC 服務器：**
- 需與加工機床在同一網絡（用於遠程監控功能）
- 需要機床端安裝并配置 VNC 服務

**機床配置文件：**
- `MachineData.mac` 文件需正確設置
- 文件位置：軟件安裝目錄下的機床類型文件夾

## 1.4 授權與啟動說明

### 1.4.1 授權機制

MyCAM 采用授權驗證機制，軟件啟動時會自動進行授權檢查。

**授權檢查流程：**

```
軟件啟動
    ↓
授權驗證
    ↓
驗證授權狀態
    ↓
[授權有效] → 軟件正常啟動
    ↓
[授權無效] → 顯示錯誤信息並自動關閉
```

> **⚠️ 注意事項：**
> - 如出現授權錯誤，請聯系系統管理員或軟件供應商
> - 確保網絡連接正常（如使用網絡授權）

### 1.4.2 軟件啟動

**步驟 1：檢查前置條件**
- 確認 .NET Framework 4.8 已安裝
- 確認顯卡驅動程序已更新

**步驟 2：啟動軟件**
- 雙擊執行 `MyCAM.exe`
- 系統將自動進行授權驗證

**步驟 3：初始化檢查**

軟件啟動後會進行以下初始化：

**1. Viewer 初始化**
- 如初始化失敗，會顯示信息："初始化 Viewer 錯誤，請檢查顯卡是否已安裝驅動"
- 請確認 OpenCascade 庫已正確安裝

**2. 機床數據載入**
- 系統會自動讀取 `MachineData.mac` 機床配置文件
- 如讀取失敗，系統會使用默認機床參數，並顯示警告信息

**3. 機床模型載入**
- 根據機床類型（Table / Mix / Spindle）載入對應的 3D 模型文件
- 模型文件位置：軟件安裝目錄下的機床類型文件夾

> **📸 截圖建議 1-3：啟動界面 - 軟件成功啟動後的初始界面，顯示各功能區已正常載入**  
> **對應文件名稱：** `ch01_03_啟動界面.png`  
> **重要性：** ⭐⭐ 建議

### 1.4.3 常見啟動問題

| 問題 | 可能原因 | 解決方案 |
|------|---------|---------|
| 授權驗證失敗 | 授權過期或無效 | 聯系軟件供應商更新授權 |
| Viewer 初始化錯誤 | 顯卡驅動未安裝 | 更新顯卡驅動程序 |
| 機床數據載入失敗 | 配置文件丟失 | 檢查 MachineData.mac 文件 |
| .NET 錯誤 | Framework 版本過低 | 安裝 .NET Framework 4.8 |
